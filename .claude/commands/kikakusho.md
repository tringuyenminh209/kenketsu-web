# 企画書ワンシート生成

日本式ワンシート企画書（A4・1枚・PDF印刷対応）のHTMLファイルを生成する。

## デザイン原則
- 背景：白、文字：黒 — カラーなし
- セクションヘッダー：`■ タイトル`、左border黒、背景 #f0f0f0
- ボックス：実線または破線の黒・グレー枠
- 矢印フロー：`→` または `⇒` テキスト文字
- フォント：Yu Gothic / Meiryo / Hiragino（日本語優先）
- 統計数値：大きいフォント（18〜22px）、ラベル小文字（7〜8px）
- 表：`border-collapse: collapse`、ヘッダー背景 #e8e8e8
- ガントチャート：`background:#333` の `div` バー
- フッター：`position:fixed; bottom:6mm` にページ番号とドキュメントID

## PDF印刷時の背景色対策（必須）
ChromeはデフォルトでCSS背景色を印刷時に無効化する。
以下を **必ずすべてのbody・背景色を持つ要素** に付与すること：

```css
-webkit-print-color-adjust: exact;
print-color-adjust: exact;
```

適用対象：`body`、`.sec-head`、`th`、`.gantt-bar`、背景色付きの全要素。
`body` に付与すれば子要素に継承されるが、念のため個別にも指定する。

## ファイル要件（システム設計授業）
- サイズ：A4（210mm × 297mm）
- 提出：PDF（ChromeでCtrl+P → PDFとして保存）
- ファイル名形式：`26IDS0-0[出席番号]-P[課題番号].pdf`
- ヘッダー：`26IDS0-0[出席番号]-P[課題番号]　森彰子 先生 様` ＋ 日付
- フッター：ドキュメントID ＋ ページ番号（`- 1 -`）
- 氏名：`SK3A　NGUYEN MINH TRI`（出席番号22）

## 標準セクション構成
1. **企画の背景（現状分析）** — 統計数値3つ ＋ 説明文
2. **現状分析・企画のゴール** — 問題リスト `⇒` 課題ボックス ＋ ゴールボックス
3. **コンセプト** — 2カラム（キャッチコピー破線枠 ＋ 説明文）
4. **解決策・システム概要** — 2カラムボックス（A案・B案）
5. **利用フロー** — 5ステップ矢印フロー
6. **期待される効果** — 4行テーブル（指標・効果・根拠）
7. **スケジュール ＋ 費用** — 2カラム（ガントテーブル ＋ 料金テーブル）

## 使い方
ユーザーが企画内容（テーマ、キーワード、ターゲット、問題、解決策）を提示したら、
上記デザイン原則・セクション構成に従って完全なHTMLファイルを生成し、
`C:\Users\2240788\OneDrive - yamaguchigakuen\デスクトップ\３年\システム設計\[第X回]\` に保存する。

出力後は「ChromeでCtrl+P → PDFとして保存」の手順をリマインドする。
