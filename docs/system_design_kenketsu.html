■ 1. システム概要
📌 本サイトはECC社会貢献センターが毎年継続使用する想定で設計。担当学生の卒業後も先生方のみで運用可能。
項目内容
システム名献血イベントWebサイト（kenketsu-web）
目的献血ボランティアイベントの告知・参加申込・アンケート収集（毎年更新して使用）
主な利用者学生（参加申込・アンケート）、ECC社会貢献センター教員（管理・データ確認）
規模Small（1〜2人開発・静的サイト・年次更新型）
開発モデルウォーターフォール（初回）→ 以降は年次メンテナンス
言語日本語
年次更新方法src/config/event.ts を編集してGitHub push → Vercel自動デプロイ（3分）

■ 5. 多言語対応（i18n）
📌 ECC専門学校には日本語以外の留学生が多いため、4言語に対応する。
言語コード言語対象ユーザーフォント（Google Fonts）備考
ja日本語日本人学生・先生Noto Sans JPデフォルト言語
myမြန်မာဘာသာ（ミャンマー語）ミャンマー人留学生Noto Sans Myanmar
neनेपाली（ネパール語）ネパール人留学生Noto Sans Devanagari
zh中文简体（中国語簡体）中国人留学生Noto Sans SC
実装方針
項目内容
ライブラリi18next + react-i18next
翻訳ファイルsrc/locales/ja.json（マスター）、my.json、ne.json、zh.json
言語セレクターNavbarに配置。選択言語は localStorage に保存し次回アクセス時も維持
フォント切替<html lang> 属性を言語コードに同期 → CSSで言語別フォントを適用
Admin画面日本語固定（管理者は先生のみ）
コーディングルールUIテキストは必ず t('key') 経由。ハードコード禁止

■ 8. データベース設計
8.1 テーブル一覧
テーブル名（論理）テーブル名（物理）種別説明

アンケート回答survey_responsesトランWebサイト改善用アンケートの回答データ。event_yearで年度別管理。
8.2 registrations テーブル定義
項番列名（論理）列名（物理）属性データ型桁数PKNNDefault備考
1IDid文字UUID—●●gen_random_uuid()主キー
2年度event_year数値INTEGER—●—★新規 例: 2026
3学生番号student_id文字VARCHAR20●—例: SK3A-001
4名前name文字TEXT—●—フルネーム
5クラスclass文字VARCHAR20●—例: SK3A
6登録日時created_at日時TIMESTAMPTZ—●now()自動設定
8.3 survey_responses テーブル定義
項番列名（論理）列名（物理）属性データ型桁数PKNNDefault備考
1IDid文字UUID—●●gen_random_uuid()主キー
2年度event_year数値INTEGER—●—★新規
3満足度評価rating数値INTEGER—NULL1〜5（CHECK制約）
4情報有用フラグfound_info_useful真偽BOOLEAN—NULL
5参加意欲フラグwould_participate真偽BOOLEAN—NULL
6自由記述comment文字TEXT—NULL任意
7回答日時created_at日時TIMESTAMPTZ—●now()自動設定
8.4 RLSポリシー（Row Level Security）
テーブル操作許可ロール条件
registrationsINSERTanonymous（全員）WITH CHECK (true)
registrationsSELECTauthenticated（先生）auth.role() = 'authenticated' ★新規
survey_responsesINSERTanonymous（全員）WITH CHECK (true)
survey_responsesSELECTauthenticated（先生）auth.role() = 'authenticated' ★新規
📌 管理者（先生）の追加方法: Supabase Dashboard → Authentication → Users → Invite user でメールアドレスを登録するだけ。コード変更不要。

■ 10. 初回セットアップ手順（一度だけ）
10.1 Vercel × GitHub 連携
# 1. GitHubにkenketsu-webリポジトリを作成してpush
git remote add origin https://github.com/[username]/kenketsu-web.git
git push -u origin main

# 2. vercel.json をプロジェクトルートに作成（SPAルーティング用）
# { "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }

# 3. Vercel.com → Add New Project → Import kenketsu-web
# 4. Environment Variables に VITE_SUPABASE_URL と VITE_SUPABASE_ANON_KEY を設定
# 5. Deploy → 完了（以降はpushするたびに自動デプロイ）
10.2 Cloudflare DNS設定
タイプ名前コンテンツプロキシ備考
CNAMEkenketsucname.vercel-dns.comオン（橙雲）Vercel提供のCNAME
10.3 Supabase 管理者ユーザー登録
# Supabase Dashboard → Authentication → Users → Invite user
# 先生のメールアドレスを入力 → 招待メールが届く → パスワード設定
# 以降は /admin/login からログイン可能
■ 11. 年次更新手順（先生 or サポート学生が実施）
📌 毎年この手順だけでOK。サーバー操作・コマンド実行は不要。
Step担当作業内容詳細
1 サポート学生
または先生		 event.tsを編集 GitHubのsrc/config/event.tsをブラウザ上で直接編集。年・日時・場所・定員を変更。
2 自動（Vercel） 自動デプロイ GitHubへのCommit検知 → Vercelが自動ビルド・デプロイ（約3分）。操作不要。
3 先生 サイト確認 本番URLにアクセスして日時・場所が正しく表示されているか確認。
4 先生 告知・受付開始 学生にURLを共有してイベント告知。参加申込が自動でSupabaseに蓄積される。
5 先生 データ確認・CSV出力 /admin にログイン → 年度フィルターで当年を選択 → 「CSVエクスポート」ボタンを押す。
11.1 event.ts の変更箇所（毎年）
// src/config/event.ts — ここだけ毎年変更する
export const EVENT_CONFIG = {
  year: 2027, // ← 年度を変更
  title: "献血ボランティア活動",
  date: "2027年9月○日（○）", // ← 日付を変更
  time: "10:00 〜 16:00",
  location: "○○キャンパス ○○号館 1F", // ← 場所を変更
  capacity: 50,
  organizer: "ECC社会貢献センター",
} as const
